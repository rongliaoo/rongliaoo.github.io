Leetcode 300. Longest Increasing Subsequence

300. Longest Increasing Subsequence

題意

Given an integer array nums, return the length of the longest strictly increasing subsequence.
給定一個整數陣列nums，回傳嚴格遞增的最長序列。

Constraints:

  • 1 <= nums.length <= 2500
  • -104 <= nums[i] <= 104

Solution 1 - Dynamic programming

思路：

  1. 建立一個 dp 陣列，其中 dp[i] 表示以 nums[i] 結尾的最長嚴格遞增子序列的長度。
  2. 遍歷陣列，對於每一個 nums[i]，找出所有比 nums[i]（代表當前的數值） 小的 nums[j]（代表先前的數值），並更新 dp[i] 為 dp[j] + 1 的最大值。
  3. 最後的答案為 dp 陣列中的最大值。

Solution 2: Greedy with Binary Search

考慮nums = [2, 6, 8, 3, 4, 5, 1]，建立第一個子序列sub1 = []

  1. 放入第一個元素sub1 = [2]
  2. 6，與前一元素比較2 < 6，因此放入sub1 = [2, 6]
  3. 8，與前一元素比較6 < 8，因此放入sub1 = [2, 6, 8]
  4. 3，與前一元素比較8 > 3，不能放入sub1，但因為在訪問到後面的序列時有可能建構其他子序列，宣告另一個子序列並放入sub2 = [2, 3]
  5. 4，與sub1最後元素8 > 4，不能放入sub1，與sub2最後元素3 < 4，因此放入sub2 = [2, 3, 4]
  6. 5，與sub1最後元素8 > 5，不能放入sub1，與sub2最後元素4 < 5，因此放入sub2 = [2, 3, 4, 5]
  7. 1，與sub1最後元素8 > 1，不能放入sub1，與sub2最後元素5 > 1，因此放入新子序列sub3 = [1]
  8. 最終sub1 = [2, 6, 8]、sub2 = [2, 3, 4, 5]、sub3 = [1]，其得到最長嚴格遞增子序列為sub2，長度為4

Key point

在上面的步驟中，需要一直保留與建立不同的 sub 陣列 （sub1， sub2...， subk），這會導致性能不佳。但是注意到，可以只保留一個 sub 陣列，當新數位 x 不大於 subsequence sub 的最後一個元素時，我們進行二分搜索，找到 sub 中的最小元素 >= x，並替換為數位 x。

範例運行：nums = [2, 6, 8, 3, 4, 5, 1]

  1. 將第一個數字加入子序列 sub，sub = [2]

  2. 6大於目前子序列的最後一個數字 2，因此將其新增到 sub 的末尾：sub = [2, 6]

  3. 8大於目前子序列的最後一個數字 6，因此將其新增到 sub 的末尾：sub = [2, 6, 8]

  4. 3小於目前子序列的最後一個數字 8，因此無法直接延長子序列。

    • 需要找到 sub 中第一個大於或等於 3 的數字（使用二分搜尋），該數字為6。
    • 用 3替換6，結果為：sub = [2, 3, 8]

  5. 4小於目前子序列的最後一個數字 8，因此無法直接延長子序列。

    • 需要找到 sub 中第一個大於或等於4 的數字（使用二分搜尋），該數字為 8。
    • 用 4 替換 8，結果為：sub = [2, 3, 4]

  6. 5大於目前子序列的最後一個數字 4，因此將其新增到 sub 的末尾sub = [2, 3, 4, 5]

  7. 1小於目前子序列的第一個數字 2，因此無法直接延長子序列。

    • 需要找到 sub 中第一個大於或等於 1 的數字（使用二分搜尋），該數字為 2。
    • 用 1 替換 2，結果為：sub = [1, 3, 4, 5]

  8. 最終結果：sub 的長度即為最長遞增子序列的長度，在此範例中為：len(sub) = 4

C++ lower bound用法

  • start: 搜尋範圍的起始迭代器。

  • end: 搜尋範圍的結束迭代器（不包含在範圍內）。

  • value: 要搜尋的值。
    返回值是一個指向搜尋結果的 迭代器：

  • 若找到一個 大於或等於 value 的元素，返回指向該元素的迭代器。

  • 若所有元素都小於 value，則返回 end。
