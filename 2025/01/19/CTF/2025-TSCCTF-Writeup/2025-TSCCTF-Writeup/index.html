Web

Be_IDol

一開始進入是一個需要登入的畫面，經由確認不是SQLi，轉而找web中的資訊，F12看到有PHPSESSID=secretbackdoor123，在Application換掉後成功登入

登入後是一個內部文件管理系統，嘗試下載檔案之後發現檔案可以下載但是不能正常打開

確認一下檔案內容，發現是其他型態資料，more查看訊息顯示是normal的，根據題意猜測應該是要找不正常的檔案

就寫了一個腳本能夠wget 10000~12000的檔案並且more查看載下來的檔案有沒有怪怪的訊息

結果沒發現頁面顯示的檔案編號只有到11000，但是到11001出現了不一樣的訊息，本來拿去一開始登入頁面想說可不可以登入，但無法，所以推測是要找出11000之後的奇怪檔案，因此繼續跑腳本

一直到12001，到網址http://172.31.0.2:8057/download.php?file_id=12001 發現是shell，指令find / -name 'flag' -print 2>/dev/null發現一個沒有限制權限的flag目錄

查看之後發現flag.txt

cat 之後得到flag

Flag: TSC{You_can_be_ID0R_12353oujhefrgiuoewihoqweihfo}

MISC

baby jail

參考https://hackmd.io/@sdHUgwi_RuWXfWL7njKG3A/BkGRIJKsi

不能用import、builtins，利用繼承物件尋找函式列表。
().__class__.__base__.__subclasses__()
object類下去尋找所也得子類別，尋找可用的類別，因為沒有讀取檔案的類別（io.FileIO），因此轉而尋找遠端進入shell函式。

payload

A_BIG_BUG

「伺服器訊息區塊 (SMB)」通訊協定是一種主從架構通訊協定，用於對網路上的檔案、目錄、印表機、序列埠及其他資源進行共用存取。 它還提供經過鑑別的程序間通訊 (IPC) 機制。（smb）
題目提供instance，還有smp port，利用smbclient來分析smb協議遠程漏洞
smbclient -L 172.31.0.2 -p [sbmport]

smbclient //172.31.0.2/uploads -p 30293 -U ctfuser

到/uploads目錄下查看並沒有可以利用的東西，IPC目錄也是
smbclient //172.31.0.2/IPC$ -p 30293 -U ctfuser

1/17

原來是可以自己上傳shell…
可惡
利用指令netexec smb 172.31.0.2 --port 30185 -u ctfuser --shares可以掃出共享資料的權限，發現/uploads是可以寫的
所以上傳webshell
shell.php

訪問http://172.31.0.2:<題目給的http ip>/uploads/shell.php
輸入find / -type f -iname "*flag*" 2>/dev/null
